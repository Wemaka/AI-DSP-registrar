# 🚂 Умный ДСП-Регистратор

**Мультимодальный AI-ассистент дежурного по станции (ДСП)**  
Курсовая работа: «Разработка мультимодального чат-бота для автоматизации ведения журнала движения поездов»

---

## 📋 Соответствие требованиям

| № | Требование | Реализация |
|---|---|---|
| 1 | Web-приложение | Streamlit (`app.py`) |
| 2 | Элементы управления | Кнопки, вкладки, слайдеры, загрузка файлов, toggle |
| 3 | Ввод/генерация текста | st.chat_input, LLM (GPT-4o-mini), цепочки |
| 4 | Аудио-модальность | Whisper STT + gTTS TTS |
| 5 | Внешние данные | schedule.csv (таблица) + PDF через RAG + DuckDuckGo (интернет) |
| 6 | Контекстное окно | `ConversationBufferWindowMemory` (LangChain) |
| 7 | ≥2 цепочки | Парсер сущностей + Генератор инструкций |
| 8 | RAG | FAISS + sentence-transformers + нормативы РЖД |
| 9 | ≥2 агента | Агент-Регистратор + Агент-Технический консультант |
| 10 | ML-модель | TF-IDF + LogisticRegression (scikit-learn) |

## 🚀 Установка и запуск

### 1. Клонируйте / распакуйте проект

```bash
cd dsp_assistant
```

### 2. Создайте виртуальное окружение

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

> **Примечание по Whisper:** для работы STT также нужен `ffmpeg`:
> - Windows: https://ffmpeg.org/download.html
> - Linux: `sudo apt install ffmpeg`
> - Mac: `brew install ffmpeg`

### 4. Настройте API ключ

```bash
cp .env.example .env
# Отредактируйте .env, вставьте ваш OpenAI API ключ
```

Или введите ключ прямо в интерфейсе приложения.

### 5. Запустите приложение

```bash
streamlit run app.py
```

Откроется браузер по адресу `http://localhost:8501`

## ⚙️ Архитектура системы

### Поток обработки голосовой команды

```
Голос ДСП
  → [Whisper STT] → Текст
  → [ML-классификатор] → Класс события (0/1/2)
  → [Цепочка 1] Парсер сущностей → JSON {поезд, путь, статус}
  → [RAG] Поиск в нормативах РЖД
  → [Цепочка 2] Генератор инструкций → Текст для ДСП
  → [Агент-Регистратор] → journal.csv
  → [gTTS] Синтез ответа → Воспроизведение
```

### ML-классификатор (Требование 10)

- **Алгоритм:** TF-IDF (char_wb, 2-4 граммы) + LogisticRegression
- **Датасет:** 70 примеров, 3 класса, ручная разметка
- **Метрика:** CV F1-macro ≈ 0.92
- **Преимущество:** работает без API-ключа, мгновенный отклик

| Класс | Описание | Действие системы |
|---|---|---|
| 0 — Штатная | Прибытие, отправление | Запись в журнал |
| 1 — Тех. сбой | Стрелки, светофоры | Запись + инструкции |
| 2 — АВАРИЯ | Сход, пожар, излом рельса | Красный баннер + немедленные инструкции |

### LangChain-цепочки (Требование 7)

**Цепочка 1 — Парсер сущностей:**
```
Промпт (System) + Текст (Human)
  → LLM (GPT-4o-mini)
  → JSON: {train_id, track, status, location, issue, action_required}
```

**Цепочка 2 — Генератор инструкций:**
```
Ситуация + Класс + RAG-контекст
  → LLM (GPT-4o-mini)
  → Пронумерованные инструкции для ДСП
```

### Агенты (Требование 9)

**Агент 1 — Регистратор:**
- Инструменты: `write_journal_entry`, `read_journal`, `journal_stats`, `check_schedule`
- Задача: вносить и читать записи журнала

**Агент 2 — Технический консультант:**
- Инструменты: `search_regulations` (RAG), `search_internet` (DuckDuckGo), `explain_signal`
- Задача: отвечать на технические вопросы ДСП
